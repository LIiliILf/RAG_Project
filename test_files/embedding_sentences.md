# Embedding 测试句子

这份文件用于第 6 节 Embedding 实验。

目标是观察：

```text
1. 文本能否转换成向量。
2. query 和 document 的语义相似度是否符合直觉。
3. 语义相近但关键词不同的句子是否能排到前面。
4. 明显无关的句子是否不会被错误匹配。
```

## documents

1. 文档解析负责把 PDF、Word、Markdown 和 TXT 文件转换成纯文本。
2. 文本分块会把长文本切成适合检索的小片段。
3. Embedding 可以把文本转换成向量，让机器计算语义相似度。
4. FAISS 可以把文档向量存起来，并支持本地相似度检索。
5. Prompt 构建会把用户问题和检索到的上下文组合起来。
6. RAG 系统会先检索资料，再让大模型基于上下文回答问题。
7. 今天天气很好，适合去公园散步。
8. 晚餐可以选择米饭、青菜和鸡蛋。

## queries

1. 如何把文件内容变成可以处理的文本？
2. 怎么把一篇很长的文档拆成小段？
3. 怎样把句子变成向量？
4. 哪个组件负责保存向量并做相似度搜索？
5. RAG 为什么要先查资料再回答？
6. 今天适合出门走走吗？

## observation_notes

后续运行 embedding 实验时，请重点观察：

```text
1. query 是否能匹配到 expected_matches 中的预期 document。
2. RAG 相关 query 是否会错误匹配到天气或晚餐句子。
3. “文件内容变成文本”和“PDF、Word 转换成纯文本”这种关键词不同但语义相近的句子是否能靠近。
4. 如果中文匹配效果不好，后续是否需要换中文 embedding 模型。
```
