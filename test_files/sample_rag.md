# RAG 文档解析测试文件

固定关键词：`RAG_TEST_KEYWORD_2026`

这是一份用于测试 `document_loader.py` 的 Markdown 文件。

## 核心概念

RAG 的流程可以分为：

1. 文档解析
2. 文本分块
3. 向量化
4. 检索
5. Prompt 构建
6. 生成回答

## 检查点

- 解析后文本是否为空。
- 固定关键词是否还在。
- 中文内容是否出现乱码。
- Markdown 标题、列表、表格是否以纯文本形式保留。

## 表格样例

| 格式 | 是否支持 | 备注 |
| --- | --- | --- |
| TXT | 是 | 最适合做最小测试 |
| Markdown | 是 | 标题和列表会以纯文本保留 |
| Word | 是 | 依赖 python-docx |
| PDF | 是 | 扫描版 PDF 可能解析不出来 |
